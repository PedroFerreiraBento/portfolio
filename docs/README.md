# Documentação – Caos Domado

Esta pasta centraliza a **documentação do portfólio Caos Domado**.

## Arquivos principais

- **Visão de produto e frontend**  
  Arquivo atual: `../CAOS_DOMADO_PORTIFOLIO.md`  
  Contém a visão completa do portfólio, princípios de design, arquitetura de frontend, modelo de dados mockado, UX por fluxo e roadmap de implementação.

- **Documentação de rotas**  
  Arquivo atual: `../CAOS_DOMADO_ROUTES.md`  
  Descreve todas as rotas da SPA (paths, componentes, parâmetros, relação com i18n e fluxos de navegação).

## Organização sugerida

Se quiser deixar tudo fisicamente dentro de `docs/`, você pode, no seu sistema de arquivos/IDE:

- Mover `CAOS_DOMADO_PORTIFOLIO.md` para `docs/CAOS_DOMADO_PORTIFOLIO.md`.
- Mover `CAOS_DOMADO_ROUTES.md` para `docs/CAOS_DOMADO_ROUTES.md`.

A partir daí, este repositório passaria a ter a seguinte estrutura de documentação:

```text
Portifolio/
  docs/
    CAOS_DOMADO_PORTIFOLIO.md
    CAOS_DOMADO_ROUTES.md
    README.md
```
